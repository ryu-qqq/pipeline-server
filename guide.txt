  ### 2단계: 누락 테스트 보완                 
                                                                                                                                                                                                                    
  분석 결과에서 발견된 누락 케이스를 실제 테스트 코드로 작성해줘.                                                                                                                                                   
  특히:                                                                                                                                                                                                             
  - 경계값 테스트 (Temperature -90/60, Confidence 0.0/1.0 경계)                                                                                                                                                     
  - Refiner에서 모든 필드가 누락된 경우 (빈 dict)                                                                                                                                                                   
  - PhaseRunner의 INSERT IGNORE 중복 처리 흐름                                                         
  - PipelineService의 resume 로직 (last_completed_phase별 분기 전수)                                                                                                                                                
  - DataReadService/TaskReadService/RejectionReadService의 단위 테스트 (있는지 확인)                                                                                                                                
  - Router의 cursor 페이징 (after) + 에러 케이스 (page+after 동시)                                                                                                                                                  
  - AnalyzeTask 상태 전이 체인 (create → processing → complete, create → processing → fail)                                                                                                                         
                                                                                                                                                                                                                    
  ### 3단계: docs/testing-strategy.md 문서 작성                                                                                                                                                                     
                                                                                                                                                                                                                    
  아래 구조로 작성:                                                                                                                                                                                                 
                                                                                                                                                                                                                    
  1. **테스트 피라미드 개요** — 레이어별 테스트 수, 비율, 실행 시간                                                                                                                                                 
  2. **Domain 테스트** — 무엇을 검증하는가, 어떤 원칙(순수 Python, 외부 의존 없음)                     
  3. **Application 테스트** — Mock 전략, 각 서비스/리파이너별 검증 항목
  4. **Adapter 테스트** — SQLite in-memory 전략, TestClient 전략, QueryBuilder 검증 방법                                                                                                                            
  5. **통합 테스트** — testcontainers 구성, 시나리오별 검증 항목, Docker 의존성
  6. **커버리지 분석** — 커버되는 것 / 커버되지 않는 것 (명시적으로)                                                                                                                                                
  7. **테스트 실행 방법** — 단위/통합/전체 실행 명령어                                                                                                                                                              
                                                                                                                                                                                                                    
  규칙:                                                                                                                                                                                                             
  - 테스트 코드를 먼저 읽고 실제 동작을 확인한 후 문서 작성                                                                                                                                                         
  - 누락된 테스트는 코드로 먼저 작성하고, 문서에 반영                                                                                                                                                               
  - pytest를 실제로 실행해서 전체 통과 확인 후 문서 작성
