# 과정 기록서 & AI 활용 기록 관리 스킬

## 배경

이 프로젝트는 결과물뿐 아니라 **"과정"**을 중시한다.
- 어떤 사고를 거쳤는지
- AI를 어떻게 활용했는지 (단순 사용이 아닌 효과적 활용)
- AI가 생성한 내용을 그대로 쓰지 않고 본인의 판단을 더한 부분

이 스킬은 이러한 기록을 **자연스럽게 축적하고, 최종 산출물로 변환**하는 것을 돕는다.

---

## 파일 구조

```
docs/
├── seeds/                          # 시드 데이터 (날짜별 원본 기록)
│   ├── YYYY-MM-DD-decisions.md     # 의사결정 기록
│   ├── YYYY-MM-DD-ai-usage.md      # AI 활용 기록
│   └── YYYY-MM-DD-progress.md      # 진행상황 기록
├── progress-journal.md             # 최종 산출물: 과정 기록서
└── ai-usage-log.md                 # 최종 산출물: AI 활용 기록
```

---

## 모드 1: 기록 (Record)

사용자가 "기록해줘", "이거 기록", "decision 기록" 등으로 호출하거나,
대화 중 의사결정/AI 활용이 감지되면 시드 데이터를 저장한다.

### 의사결정 기록 (`docs/seeds/YYYY-MM-DD-decisions.md`)

현재 대화 맥락에서 의사결정을 추출하여 기록한다.
기존 파일이 있으면 하단에 추가(append), 없으면 새로 생성한다.

포맷:
```markdown
## Decision N: {제목}
- **시점**: {언제 이 결정을 했는지}
- **선택**: {무엇을 선택했는지}
- **대안**:
  - (A) {대안 1} — {장단점}
  - (B) {대안 2} — {장단점}
  - (C) {선택한 것} — 선택됨
- **이유**: {왜 이것을 선택했는지, 구체적 근거}
- **결과**: {이 결정의 결과}
```

핵심: **"선택한 것"만이 아니라 "대안은 뭐였고, 왜 버렸는지"**가 가치 있다.

### AI 활용 기록 (`docs/seeds/YYYY-MM-DD-ai-usage.md`)

AI 도구를 활용한 작업을 기록한다.

포맷:
```markdown
## AI-N: {작업명}
- **도구**: {사용한 도구 — Claude Code, 에이전트명 등}
- **목적**: {AI를 써서 뭘 하려고 했는지}
- **AI 제안**: {AI가 제안/생성한 내용 요약}
- **본인 판단**: {AI 제안에 대해 본인이 어떤 판단을 더했는지}
- **결과**: {최종 결과물}
```

핵심: **"AI 제안"과 "본인 판단"을 분리**하는 것.

### 진행상황 기록 (`docs/seeds/YYYY-MM-DD-progress.md`)

하루 작업 내용을 요약한다.

포맷:
```markdown
## 수행 내용
- {한 일 목록}

## 기술적 문제 해결
- **문제**: {뭐가 안 됐는지}
- **해결**: {어떻게 해결했는지}
- **배운 점**: {이 과정에서 알게 된 것}

## 에이전트 활용 현황
| 에이전트 | 작업 | 산출물 | FIX 횟수 |
|---|---|---|---|
```

## 모드 2: 생성 (Generate)

"과정 기록서 생성해줘", "AI 기록 정리해줘" 등으로 호출하면,
`docs/seeds/`의 시드 데이터를 모아서 최종 산출물을 생성한다.

### 과정 기록서 생성 (`docs/progress-journal.md`)

모든 시드를 Day별로 조합하여 정리된 포맷으로 변환한다.

```markdown
# 과정 기록서 (Progress Journal)

## Day 1 - 2026-04-09 [인프라 + 에이전트 시스템 구축]

### 수행 내용
- (seeds/progress에서 추출)

### 의사결정
- (seeds/decisions에서 추출, 요약)

### 기술적 문제 해결
- (seeds/progress에서 추출)

### 에이전트 활용
- (seeds/progress에서 추출)
```

### AI 활용 기록 생성 (`docs/ai-usage-log.md`)

```markdown
# AI 활용 기록

## 활용 개요
- 사용 도구: Claude Code (CLI), 멀티 에이전트 시스템
- 활용 방식: 코드 생성(Builder), 설계 검증(Reviewer), 컨벤션 준수(Guardian)

## 상세 기록

### 1. {작업명}
- **활용 도구**: ...
- **AI의 역할**: ...
- **본인의 판단**: ...
- **효과**: ...

(seeds/ai-usage에서 추출, 정제)
```

## 사용 흐름

### 작업 중 (기록)
작업하다가 의사결정을 내렸거나, AI를 활용한 작업이 끝나면:
1. 사용자가 "이거 기록해줘" 또는 자동 감지
2. 현재 대화 맥락에서 의사결정/AI 활용 내역 추출
3. 해당 날짜의 시드 파일에 append

### 하루 끝 (정리)
하루 작업이 끝나면:
1. "오늘 기록 정리해줘"
2. 오늘 날짜의 시드 파일들을 읽고 진행상황 시드 생성/보강

### 제출 전 (생성)
과제 제출 전:
1. "과정 기록서 생성해줘" / "AI 활용 기록 생성해줘"
2. 모든 시드를 조합하여 최종 문서 생성

## 하네스 연동

각 하네스가 실행될 때 자동으로 기록 가능한 정보:

| 하네스 | 자동 기록 대상 |
|---|---|
| domain-harness | FIX 횟수, ESCALATION 여부, 리뷰 등급 |
| service-harness | CQRS 분리 결정, 트랜잭션 경계 결정 |
| persistence-harness | 인덱스 설계 결정, Polyglot 저장소 선택 |
| infra-harness | 캐시 전략 결정, Docker 구성 결정 |
| pipeline | 전체 Phase 진행 현황, 소요 시간 |

---

## 주의사항

- 시드는 **원본 기록**이므로 수정보다는 추가(append)를 우선한다
- 최종 문서 생성 시 시드를 그대로 복붙하지 않는다. 읽기 좋게 정제하되 핵심 정보는 유지
- 의사결정에서 "대안"은 최소 2개 이상 기록한다. 선택지가 1개뿐이면 의사결정이 아니다
- AI 활용에서 "본인 판단" 필드가 비어있으면 안 된다
- 에이전트가 수행한 것과 사용자가 판단한 것을 명확히 구분하여 기록
